# Repository Review Rules

- Prioritize correctness over style
- Focus on meaningful findings only:
  - bugs
  - missing tests
  - maintainability risks
  - security concerns
- Ignore purely cosmetic formatting issues
- Suggest tests when business logic changes
- Prefer concise, actionable feedback tied to changed files
- Avoid speculative comments unless confidence is low and clearly stated
- Do not recommend broad rewrites unless the diff obviously creates systemic risk
